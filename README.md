# ann-feedback

링크드인 글쓰기에 대한 에디터의 인사이트를 제공하는 Claude Code 스킬.

글을 붙여넣으면 **근본 진단 → 인사이트 카드 → 수정 루프**를 거쳐, 발행 가능한 글이 될 때까지 함께합니다.

## 설치

```bash
claude install-skill https://github.com/annnzel/ann-feedback
```

## 사용법

Claude Code에서:

```
/ann-feedback
```

또는 자연어로:
- "글 봐줘"
- "피드백 줘"
- "인사이트 카드 만들어줘"

글을 붙여넣으면 분석이 시작됩니다.

---

## 어떻게 작동하나요?

### 1. 근본 진단 — 6 Lens보다 먼저

글을 받으면 세부 분석 전에 **두 가지 질문**을 먼저 던집니다:

- **"감동, 재미, 신선한 관점, 적용점 중 뭐가 있는가?"**
- **"이 글을 누가 쓴 건지 알 수 있는가?"**

이 답에 따라 진단 레벨이 결정됩니다:

| 레벨 | 의미 | 이후 피드백 |
|------|------|-------------|
| 🔴 **갈아엎기** | 글의 방향 자체를 바꿔야 합니다 | 6 Lens 생략. 새로운 글 방향 제안 |
| 🟡 **큰 수술** | 뼈대를 살리되, 구조를 크게 바꿔야 합니다 | 6 Lens + 구조 재설계 제안 |
| 🟢 **다듬기** | 기본은 좋습니다. 부분 수정으로 더 좋아집니다 | 6 Lens + 세부 제안 |

### 2. 인사이트 카드

진단 레벨에 맞는 포맷으로 피드백을 출력합니다.

🔴일 때는 기존 글을 고치지 않습니다. 대신 **같은 소재로 완전히 다른 글의 방향**을 제안합니다.

🟡/🟢일 때는 **6 Lens 분석**(Hook, Structure, Voice, Tension, Takeaway, Engagement)과 구체적인 수정 제안을 합니다.

### 3. 수정 루프

인사이트 카드를 보고 수정한 뒤 다시 붙여넣으면:
- 이전 진단과 **변화를 추적**합니다
- 진단 레벨을 다시 매깁니다 (🔴→🟡→🟢)
- 발행 가능하다고 판단되면 **"✅ 이 글은 발행해도 됩니다"**를 알려줍니다

---

## 6 Lens 분석

| Lens | 보는 것 |
|------|---------|
| **Hook** | 첫 줄이 스크롤을 멈추게 하는가? |
| **Structure** | 글의 뼈대가 독자를 끝까지 이끄는가? |
| **Voice** | 이 글이 글쓴이만의 목소리를 갖고 있는가? |
| **Tension** | 끝까지 읽어야 할 이유가 있는가? |
| **Takeaway** | 다 읽고 한 줄로 기억되는가? |
| **Engagement** | 댓글을 유도하는 장치가 있는가? |

---

## 핵심 원칙

모든 분석의 첫 번째 질문:

> **"이 글에 당신이 있는가?"**

이름을 가리고 읽어도 누가 쓴 글인지 알 수 있어야 합니다.
누가 써도 어색하지 않은 글이면, 아무리 잘 써도 그 글은 실패입니다.

## 출력 예시

### 🔴 갈아엎기

```
## "록히드마틴 CEO라면, 창업 13년 스타트업에 $20B 계약을…"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 갈아엎기

✦ 이 글의 근본 문제
  → 좋은 뉴스 분석이다. 그런데 테크 미디어 기자가 쓴 글이다.
    "왜 이 사람이 이 글을 쓰는가?"에 대한 답이 없다.

✦ 이 글에 없는 것
  → 감동: 없다. 재미: 없다. 신선한 관점: 약하다. 적용점: 없다.

✦ 이 소재로 쓸 수 있는 다른 글
  → "Anduril 뉴스를 보고 소름이 돋았다. 내가 매일 보는 산업에서
    똑같은 일이 벌어지고 있기 때문이다."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💬 위 방향 중 하나를 골라서 다시 써보시고, 수정본을 붙여넣어 주세요.
```

### 🟡 큰 수술 / 🟢 다듬기

```
## "5일 동안 AX 여정에서 배운 것"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟡 큰 수술

✦ 이 글이 아쉬운 이유
  → Voice는 확실하다. 그런데 감정이 없다.
    5일의 여정에서 설계자가 느낀 불안이나 갈등이 안 보인다.

✦ Editor's Pick — 가장 강한 문장
  → "당연하다. 불편이 다르니까."
  → 이 2단어 선언이 이 글쓴이의 시그니처.

✦ 제안
  → [필수] Day 3의 구체적 장면 하나를 추가
  → [권장] 중반부 반복 구간 정리

✦ 6 Lens 요약
  Hook: ⬆️ 구체적 숫자
  Structure: ⚠️ 중반 반복
  Voice: ⬆️ 선언적 톤 일관
  Tension: ⚠️ 감정 부재
  Takeaway: ⬆️ 압축력
  Engagement: ❌ 장치 없음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💬 위 제안을 반영해서 수정하시고, 수정본을 붙여넣어 주세요.
```

## 이런 것은 하지 않습니다

- 맞춤법/문법 교정
- 글을 대신 써주기
- "잘 쓰셨습니다" 같은 빈 칭찬
- "해시태그를 3~5개 쓰세요" 같은 일반 팁

**솔직하게 말합니다.** 좋은 건 좋다, 아쉬운 건 아쉽다. 그리고 항상 "왜"를 설명합니다.
