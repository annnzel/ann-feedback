# ann-feedback

링크드인 글쓰기에 대한 에디터의 인사이트를 제공하는 Claude Code 스킬.

글을 붙여넣으면 **Editor's Lens 6가지 렌즈**로 분석하고, 즉시 적용 가능한 **인사이트 카드**를 생성합니다.

## 설치

```bash
claude install-skill https://github.com/annnzel/ann-feedback
```

## 사용법

Claude Code에서:

```
/ann-feedback
```

또는 자연어로:
- "글 봐줘"
- "피드백 줘"
- "인사이트 카드 만들어줘"

글을 붙여넣으면 분석이 시작됩니다.

## 무엇을 분석하나요?

### 6 Lens 분석

| Lens | 보는 것 |
|------|---------|
| **Hook** | 첫 줄이 스크롤을 멈추게 하는가? |
| **Structure** | 글의 뼈대가 독자를 끝까지 이끄는가? |
| **Voice** | 이 글이 글쓴이만의 목소리를 갖고 있는가? |
| **Tension** | 끝까지 읽어야 할 이유가 있는가? |
| **Takeaway** | 다 읽고 한 줄로 기억되는가? |
| **Engagement** | 댓글을 유도하는 장치가 있는가? |

### 핵심 원칙

모든 분석의 첫 번째 질문:

> **"이 글에 당신이 있는가?"**

이름을 가리고 읽어도 누가 쓴 글인지 알 수 있어야 합니다. 누가 써도 어색하지 않은 글이면, 아무리 잘 써도 그 글은 실패입니다.

## 출력 예시

```
## "당신이 록히드마틴 CEO라면..."
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✦ 확실히 좋아졌다
  → "저는 그렇게 보지 않았습니다" — 이 한 줄이 글을 바꿨다.
  → 마지막 양자택일 질문이 독자에게 선택을 요구한다.

✦ Editor's Pick — 가장 강한 문장
  → "한번 어떤 조직이 특정 운영체계 위에서 돌아가기 시작하면..."
  → 비유 없이 본질을 직접 설명. 더 강하다.

✦ 제안
  → [필수] 자기 산업과 연결하는 한 단락 추가
  → [권장] 중반부 7~8단락을 반으로 줄이기

✦ 6 Lens 요약
  Hook: ⬆️⬆️ 시리즈 최강
  Structure: ⚠️ 중반 반복
  Voice: ⬆️ 분석적 톤 일관
  Tension: ⬆️ "이 지점이 무섭습니다"
  Takeaway: ⬆️ 양자택일 질문
  Engagement: ✅ 질문형 마무리

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 이런 것은 하지 않습니다

- 맞춤법/문법 교정
- 글을 대신 써주기
- "잘 쓰셨습니다" 같은 빈 칭찬
- "해시태그를 3~5개 쓰세요" 같은 일반 팁

**솔직하게 말합니다.** 좋은 건 좋다, 아쉬운 건 아쉽다. 그리고 항상 "왜"를 설명합니다.
